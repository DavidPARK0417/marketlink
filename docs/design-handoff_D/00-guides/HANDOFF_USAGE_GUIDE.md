# 디자인 전달 폴더 사용 가이드 (Handoff Folder Usage Guide)

## 1. 폴더 구조 설명

이 `design-handoff` 폴더는 디자이너(또는 퍼블리셔)가 개발자에게 최종 UI 산출물을 전달하기 위한 공간입니다.

```
design-handoff/
├── 00-guides/              # 가이드 문서 (본 문서 포함)
├── 01-design-tokens/       # 색상, 폰트, 여백 등 디자인 토큰 (JSON)
├── 02-components/          # 재사용 가능한 공통 컴포넌트 명세
├── 03-pages/               # 페이지별 최종 퍼블리싱 코드 및 명세서
│   ├── dashboard/          # 대시보드
│   ├── products/           # 상품 관리
│   └── ...
└── 05-integration-prompts/ # AI/개발자를 위한 통합 가이드 프롬프트
```

## 2. 사용 방법

### 디자이너/퍼블리셔
1. 작업이 완료된 페이지의 코드를 `03-pages/[페이지명]/code.tsx`에 저장합니다.
2. 해당 페이지의 디자인 의도, 색상 코드, 인터랙션 정의를 `SPEC.md`에 작성합니다.
3. `DESIGN_UPDATE.md` (프로젝트 루트)에 변경 이력을 남깁니다.

### 개발자
1. 구현하려는 페이지의 `SPEC.md`를 먼저 읽어 UI 요구사항을 파악합니다.
2. `code.tsx`의 코드를 복사하여 실제 프로젝트(`app/wholesaler/...`)로 가져갑니다.
3. `dummy-data` 부분을 실제 백엔드 데이터 연동 로직으로 교체합니다.
4. 스타일이나 레이아웃이 깨지는 경우 `SPEC.md`의 정의를 기준으로 수정합니다.

## 3. 업데이트 규칙
- 디자인 변경이 발생하면 반드시 이 폴더의 내용을 먼저 수정한 후, 개발자에게 알립니다.
- 이 폴더는 **"Source of Truth (단일 진실 공급원)"** 역할을 해야 합니다. 실제 개발 코드가 변경되더라도, 디자인 원본은 이곳에 남아있어야 합니다.
